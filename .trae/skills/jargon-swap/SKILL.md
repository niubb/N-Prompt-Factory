# Jargon Swap - 术语轻量替换

用户在读一篇知识文章（可能是费曼笔记、技术文档等），遇到非核心的专业术语，不想花太多精力深究，只需要一个生活化的词快速替代，保证阅读不被卡住。

## 核心原则

用户当前的学习焦点是别的概念，这个术语只是"路障"，不是"目的地"。所以目标不是讲透它，而是**快速消解，让阅读继续**。

## 执行方式

### 1. 先用生活场景简短解释

不背定义，不说词源，直接用最日常的场景说清楚这个词在当前语境下是什么意思。控制在2-3句话内。

**为什么先解释再给一句话：** 直接甩一个词出来，用户没有理解的基础，不知道为什么选这个词。先给场景，用户自己就能感受到"哦，原来就是XX"，再看到一句话总结时，是确认而非猜测。

### 2. 用固定句式收尾

必须以这句话结束：

> 一句话：【术语】就是【生活化表达】。

这句话的目的是给用户一个**可以拿走直接替换的词**。用户读到这里，就能把这个生活化词代入回原文，继续往下读。

### 3. 输出后确认循环

输出完解释和一句话总结后，必须用 ask_user 询问：

> 要修改，还是保存为笔记？

用户有三种可能回应：

- **保存**：直接调用 ima-note 技能的 push_note 接口，将当前输出保存为笔记，标题为术语名
- **提修改意见**：根据用户意见调整输出内容，调整完再次用 ask_user 询问是否保存为笔记。这一步可以多轮，直到用户满意为止
- **都不需要**：结束，不保存

### 完整流程示意

1. 输出解释 + 一句话总结
2. 问：修改还是保存为笔记？
3. 如果修改 → 改完 → 再问：保存为笔记？
4. 如果保存 → 调用 push_note 保存 → 结束

### 示例

用户问：文章里出现了"头寸"，帮我用生活里的词解释一下。

**输出：**

你去菜市场摆摊，进了200斤苹果还没卖出去——这200斤苹果就是你的头寸。多头头寸就是你看涨买进了，空头头寸就是你看跌先卖了。

一句话：头寸就是你当前在市场上押的注——押了什么、押了多少、押哪边。

**然后问：** 要修改，还是保存为笔记？

用户说"押注这个词不太好，换成持仓" → 调整输出 → 再问"保存为笔记？"

用户说"保存" → 调用 push_note 保存 → 完成。

## 注意事项

- 替换词必须是**单个词或短词组**，不能是一长串描述。用户要的是能直接塞回原文的东西
- 如果术语在不同语境下有不同含义，只解释**当前文章语境下的含义**，不要展开全貌
- 不加词源、不加历史、不加延伸。那些是费曼讲解的活，这里是快拆快走
- 保存为笔记时，标题用术语名，内容就是当前输出的解释+一句话总结
